# Walkthrough: Typed clients via kubebuilder (Part 2)

In [Part 1](01-webhook-to-cr-from-scratch.md) we built `note-api` with the
**dynamic client**: an `unstructured.Unstructured{}` carrying a
`map[string]interface{}` of fields, dispatched against a hand-typed
`GroupVersionResource`. It works, but every spec field is a string key — typos
fail at runtime, refactors don't propagate, and there's no IDE help.

This part rewrites the same service against **typed** Go structs generated by
[kubebuilder](https://book.kubebuilder.io/). The CRD YAML and the Go types come
from one source: a Go file with kubebuilder markers. The api imports those types
directly and uses
[`controller-runtime`](https://pkg.go.dev/sigs.k8s.io/controller-runtime)'s
typed `client.Client`.

This is **the pattern this repo uses in production**: `internal/webhook/wizapi/`
is a stub of the wiz-operator's typed API, and `internal/webhook/executor.go`
does typed `client.Patch(ctx, &SAMLGroupMapping{...}, client.Apply, ...)` — no
`unstructured` anywhere. By the end of this walkthrough you'll have written the
same shape on a smaller scale.

---

## Why typed beats dynamic

Both clients hit the same apiserver wire format. The difference is where the
validation happens.

|                      | Dynamic (`unstructured.Unstructured`)                              | Typed (controller-runtime `client.Client`) |
| -------------------- | ------------------------------------------------------------------ | ------------------------------------------ |
| Spec fields          | `map[string]interface{}`                                           | Go struct fields                           |
| Typo in a field name | Runtime error from apiserver, sometimes silently dropped on accept | **Compile error**                          |
| Rename a field       | Grep across every consumer; hope you found them all                | Compiler enforces                          |
| IDE autocomplete     | None                                                               | Full                                       |
| DeepCopy / equality  | Hand-coded if needed                                               | Generated                                  |
| Cost to add          | Zero codegen, zero shared module                                   | An operator project to own the types       |
| Right answer when…   | You don't own the CRD, or you need to handle unknown shapes        | You own the CRD via an operator you ship   |

For a webhook receiver that always writes its own CRD, **typed wins**. The
upfront cost is a small Go module that ships the types; every consumer gets
compile-time safety in exchange.

## What changes from Part 1

We keep the `note-api` HTTP shape identical. The only changes:

1. **A sibling project, `note-operator`,** scaffolded with kubebuilder. Owns the

## Step 2 — Scaffold `note-operator`

From the parent directory of `note-api/`:

```bash
mkdir note-operator && cd note-operator

# init creates go.mod, the Makefile, the PROJECT marker, cmd/main.go, etc.
kubebuilder init \
  --domain examples.dev \
  --repo github.com/example/note-operator
```

`--domain` becomes the API group suffix. Combined with the `--group` flag in the
next command, it forms the full group name (e.g. `examples.examples.dev`). For a
tutorial we want a clean `examples.dev/v1alpha1` — pick one or the other:

```bash
kubebuilder create api \
  --group examples \
  --version v1alpha1 \
  --kind Note \
  --resource \
  --controller=false
```

When prompted:

- **Create Resource [y/n]:** y (already passed via `--resource`)
- **Create Controller [y/n]:** n (already passed via `--controller=false`)

Two important flags here:

- `--controller=false` skips scaffolding a reconciler. We only want the _types_
  and the CRD YAML — the actual reconciler (which is what an operator does for
  real) is out of scope for this walkthrough. Real operators would say
  `--controller=true` and write a `Reconcile` function in
  `internal/controller/note_controller.go`.
- `--resource` is on so we get `api/v1alpha1/note_types.go`. Without it,
  kubebuilder skips creating the Go API package.

What got generated:

```bash
tree -L 3 -I 'node_modules|bin|hack'

# ├── api
# │   └── v1alpha1
# │       ├── groupversion_info.go     # GroupVersion + SchemeBuilder

# │       └── zz_generated.deepcopy.go # auto-regen on `make generate`
# ├── cmd
# │   └── main.go                      # operator entrypoint (we won't run it)
# ├── config
# │   ├── crd
# │   │   ├── bases                    # CRD YAML lands here after make manifests
# │   │   ├── kustomization.yaml
# │   │   └── ...
# │   ├── default
# │   ├── manager
# │   ├── prometheus
# │   ├── rbac
# │   └── samples
# ├── Dockerfile
# ├── go.mod
# ├── go.sum
# ├── Makefile
# ├── PROJECT
# └── README.md
```

A lot of scaffolding. For Part 2 we **only care about** `api/v1alpha1/` and
`config/crd/bases/`. The rest is operator infrastructure we'll ignore.

> **Why scaffold a whole project just for types?** Because the convention is one
> operator project per CRD-owning team. The `note-operator` repo is the source
> of truth for the `Note` types AND the reconciler. Other consumers (the api in
> our case) import the types from
> `github.com/example/note-operator/api/v1alpha1`. This is exactly how webhookd
> consumes wiz-operator's types in `internal/webhook/wizapi/` — except we ship a
> stub there because wiz-operator hasn't published yet.

## Step 3 — Define the Note types

Open `api/v1alpha1/note_types.go`. Kubebuilder filled it with placeholder fields
(`Foo`, etc.). Replace the spec/status sections so the file looks like this:

```go
package v1alpha1

import (
	metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
)

// NoteSpec defines the desired state of Note.
type NoteSpec struct {
	// Title is a short headline for the note.
	// +kubebuilder:validation:Required
	// +kubebuilder:validation:MinLength=1
	Title string `json:"title"`

	// Body is the note's content.
	// +kubebuilder:validation:Required
	// +kubebuilder:validation:MinLength=1
	Body string `json:"body"`
}

// NoteStatus defines the observed state of Note. Empty for now —
// a real reconciler would surface conditions, observedGeneration, etc.
// We carry it through so clients can grow into it later without a
// schema break.
type NoteStatus struct{}

// +kubebuilder:object:root=true
// +kubebuilder:subresource:status
// +kubebuilder:resource:shortName=nt

// Note is the Schema for the notes API.
type Note struct {
	metav1.TypeMeta   `json:",inline"`
	metav1.ObjectMeta `json:"metadata,omitempty"`

	Spec   NoteSpec   `json:"spec,omitempty"`
	Status NoteStatus `json:"status,omitempty"`
}

// +kubebuilder:object:root=true

// NoteList contains a list of Note.
type NoteList struct {
	metav1.TypeMeta `json:",inline"`
	metav1.ListMeta `json:"metadata,omitempty"`
	Items           []Note `json:"items"`
}

func init() {
	SchemeBuilder.Register(&Note{}, &NoteList{})
}
```

Three categories of magic to notice:

1. **`+kubebuilder:object:root=true`** marks `Note` and `NoteList` as root-level
   CRD kinds. Without it the type isn't registered as something `controller-gen`
   should emit deepcopy methods for.
2. **`+kubebuilder:validation:*`** markers on the spec fields are propagated
   into the generated CRD's OpenAPI schema. The cluster apiserver will reject
   `{title: ""}` before it touches storage — server-side validation, no extra
   Go.
3. **`+kubebuilder:subresource:status`** opts the type into the `/status`
   subresource. We don't touch it in Part 2, but it's the knob that lets a
   future reconciler patch only the status without bumping
   `metadata.generation`.

`SchemeBuilder.Register(...)` in `init()` is what ties the types into the
scheme. The `groupversion_info.go` file (generated by kubebuilder) exports
`SchemeBuilder` and `AddToScheme` — that's how consumers (including our note-api
in Step 6) register the types.

`groupversion_info.go` for reference (generated, no edits needed):

```go
// +kubebuilder:object:generate=true
// +groupName=examples.dev
package v1alpha1

import (
	"k8s.io/apimachinery/pkg/runtime/schema"
	"sigs.k8s.io/controller-runtime/pkg/scheme"
)

var (
	GroupVersion  = schema.GroupVersion{Group: "examples.dev", Version: "v1alpha1"}
	SchemeBuilder = &scheme.Builder{GroupVersion: GroupVersion}
	AddToScheme   = SchemeBuilder.AddToScheme
)
```

## Step 4 — Generate the deepcopy code and the CRD YAML

Two `make` targets matter:

```bash
make generate
# → controller-gen object:headerFile=hack/boilerplate.go.txt paths=./...
# Updates api/v1alpha1/zz_generated.deepcopy.go to match note_types.go.

make manifests
# → controller-gen rbac:roleName=manager-role crd webhook paths=./...
# Updates config/crd/bases/examples.dev_notes.yaml
```

Take a look at the generated CRD:

```bash
cat config/crd/bases/examples.dev_notes.yaml
```

It mirrors what we hand-wrote in Part 1 Step 3, but with extras: `metadata`'s
OpenAPI schema, the `status` subresource declaration, the `shortName` from our
marker, and exact field ordering driven by the Go struct. **The YAML is now
derived from the Go types; you cannot let them drift.**

Apply it (replacing the Part 1 hand-rolled CRD):

```bash
# If you applied the Part 1 crd.yaml earlier, delete it first.
# kubectl delete crd notes.examples.dev

kubectl apply -f config/crd/bases/examples.dev_notes.yaml
kubectl get crd notes.examples.dev
# NAME                  CREATED AT
# notes.examples.dev    ...
```

> **Whenever you edit `note_types.go`, run `make generate manifests`.** The
> Makefile target `make` (with no args) chains lint + test + generate +
> manifests + build, so committing a mis-generated state is hard if you wire
> that into your pre-commit.

## Step 6 — Refactor `note-api` to use typed clients

Switch back to `note-api/`. Two files change: `go.mod` (dependencies) and
`main.go` (everything K8s-related).

### 6a — Wire the operator module

While `note-operator` is still local-only (no published version), use a
`replace` directive in `note-api/go.mod` so Go finds it on disk:

```bash
cd ../note-api/

# Add the import (use a placeholder version; replace points elsewhere).
go mod edit -require=github.com/example/note-operator@v0.0.0
go mod edit -replace=github.com/example/note-operator=../note-operator

# Add controller-runtime — this brings in client.Client.
go get sigs.k8s.io/controller-runtime@v0.19.0

go mod tidy
```

Once `note-operator` ships a real tagged version
(`git tag v0.1.0 && git push --tags`, then publish to a registry that Go can
reach), drop the `replace` and bump the `require` to that tag.
`go mod edit -dropreplace=github.com/example/note-operator` does it.

> **Why a `replace` directive?** Because Go modules can't `import` from a
> sibling directory directly — every import has to resolve to a Go module.
> `replace` swaps the import path's _resolution_ without changing the import
> path itself. This is also the trick used in
> [this repo's `wizapi`/operator-stub setup](../../internal/webhook/wizapi/)
> while wiz-operator is unpublished.
>
> An alternative is a [Go workspace](https://go.dev/ref/mod#workspaces)
> (`go work init && go work use ./note-api ./note-operator`) — cleaner for
> multi-module repos at the cost of an extra file. Either works.

### 6b — Build a typed client

Replace the imports + `loadKubeConfig` + `Server` setup at the top of `main.go`:

```go
package main

import (
	"context"
	"encoding/json"
	"errors"
	"fmt"
	"log"
	"net/http"
	"os"
	"time"

	metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
	"k8s.io/apimachinery/pkg/runtime"
	utilruntime "k8s.io/apimachinery/pkg/util/runtime"
	clientgoscheme "k8s.io/client-go/kubernetes/scheme"
	"k8s.io/client-go/rest"
	"k8s.io/client-go/tools/clientcmd"
	"sigs.k8s.io/controller-runtime/pkg/client"

	notev1alpha1 "github.com/example/note-operator/api/v1alpha1"
)

// scheme tells the typed client how to (de)serialize Go types ↔
// apiserver wire format. We register two groups: the standard
// Kubernetes core/built-in types (so we can Get a Namespace, etc.) and
// our Note types from the operator module.
var scheme = runtime.NewScheme()

func init() {
	utilruntime.Must(clientgoscheme.AddToScheme(scheme))
	utilruntime.Must(notev1alpha1.AddToScheme(scheme))
}

// Server now carries a typed controller-runtime client.
type Server struct {
	k8s client.Client
}

func loadKubeConfig() (*rest.Config, error) {
	cfg, err := rest.InClusterConfig()
	if err == nil {
		return cfg, nil
	}
	if !errors.Is(err, rest.ErrNotInCluster) {
		return nil, fmt.Errorf("in-cluster config: %w", err)
	}

	loadingRules := clientcmd.NewDefaultClientConfigLoadingRules()
	overrides := &clientcmd.ConfigOverrides{}
	cfg, err = clientcmd.NewNonInteractiveDeferredLoadingClientConfig(
		loadingRules, overrides,
	).ClientConfig()
	if err != nil {
		return nil, fmt.Errorf("kubeconfig: %w", err)
	}
	return cfg, nil
}

func NewServer() (*Server, error) {
	cfg, err := loadKubeConfig()
	if err != nil {
		return nil, err
	}
	c, err := client.New(cfg, client.Options{Scheme: scheme})
	if err != nil {
		return nil, fmt.Errorf("typed client: %w", err)
	}
	return &Server{k8s: c}, nil
}
```

Three things changed from Part 1:

1. **No more `noteGVR` package var.** Controller-runtime infers the GVR from the
   registered Go type — `&Note{}` is enough.
2. **A `scheme`** with both core (`clientgoscheme`) and Note types registered.
   The client refuses to Get/Create types it doesn't know about.
3. **`client.New` instead of `dynamic.NewForConfig`.** The constructor takes
   `client.Options{Scheme: scheme}`; everything else is the same `*rest.Config`
   the dynamic client took.

### 6c — Replace `handleWebhook` with typed Create

The old handler built an
`unstructured.Unstructured{Object: map[string]interface{}{...}}`. The new one
builds a `notev1alpha1.Note{}`:

```go
func (s *Server) handleWebhook(w http.ResponseWriter, r *http.Request) {
	if r.Method != http.MethodPost {
		http.Error(w, `{"error":"method not allowed"}`, http.StatusMethodNotAllowed)
		return
	}

	var req CreateNoteRequest
	if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
		http.Error(w, `{"error":"invalid JSON"}`, http.StatusBadRequest)
		return
	}
	defer r.Body.Close()

	if req.Title == "" || req.Body == "" {
		http.Error(w, `{"error":"title and body are required"}`, http.StatusBadRequest)
		return
	}

	// The whole CR is one typed Go literal. Compiler enforces every
	// field name; renaming Title in note_types.go would force this
	// caller to update too.
	note := &notev1alpha1.Note{
		ObjectMeta: metav1.ObjectMeta{
			Name:      sanitize(req.Title),
			Namespace: envOr("NAMESPACE", "default"),
		},
		Spec: notev1alpha1.NoteSpec{
			Title: req.Title,
			Body:  req.Body,
		},
	}

	ctx, cancel := context.WithTimeout(r.Context(), 5*time.Second)
	defer cancel()

	if err := s.k8s.Create(ctx, note); err != nil {
		log.Printf("create note: %v", err)
		http.Error(w, fmt.Sprintf(`{"error":"create failed: %v"}`, err), http.StatusInternalServerError)
		return
	}

	// note is mutated in place: Name, Namespace, UID, ResourceVersion,
	// CreationTimestamp are all populated post-Create.
	w.Header().Set("Content-Type", "application/json")
	w.WriteHeader(http.StatusCreated)
	_ = json.NewEncoder(w).Encode(map[string]interface{}{
		"status":    "created",
		"name":      note.Name,
		"namespace": note.Namespace,
		"uid":       note.UID,
	})
}
```

`main()`, `sanitize()`, and `envOr()` from Part 1 are unchanged.

The handler body, dynamic vs typed:

```diff
-note := &unstructured.Unstructured{
-    Object: map[string]interface{}{
-        "apiVersion": "examples.dev/v1alpha1",
-        "kind":       "Note",
-        "metadata": map[string]interface{}{
-            "name":      sanitize(req.Title),
-            "namespace": envOr("NAMESPACE", "default"),
-        },
-        "spec": map[string]interface{}{
-            "title": req.Title,
-            "body":  req.Body,
-        },
-    },
-}
+note := &notev1alpha1.Note{
+    ObjectMeta: metav1.ObjectMeta{
+        Name:      sanitize(req.Title),
+        Namespace: envOr("NAMESPACE", "default"),
+    },
+    Spec: notev1alpha1.NoteSpec{
+        Title: req.Title,
+        Body:  req.Body,
+    },
+}
```

The `Create` call:

```diff
-created, err := s.dyn.
-    Resource(noteGVR).
-    Namespace(note.GetNamespace()).
-    Create(ctx, note, metav1.CreateOptions{})
+err := s.k8s.Create(ctx, note)
```

Notice four wins:

1. **No more `apiVersion` / `kind` strings.** The Go type tells the client what
   it is.
2. **No more GVR.** Controller-runtime looks it up from the registered scheme.
3. **No more `Resource(...).Namespace(...)` plumbing.** The
   `ObjectMeta.Namespace` field on the typed object drives routing.
4. **Mutation in place.** Post-`Create`, the same `note` pointer has `note.UID`,
   `note.ResourceVersion`, `note.CreationTimestamp` populated — no need to
   thread a separate `created` return through.

If you renamed `NoteSpec.Title` to `NoteSpec.Headline` in the operator and ran
`make generate`, the api would fail to compile until the caller is updated. With
dynamic, you'd ship `"title": req.Title` straight at a confused apiserver and
only find out from the failing integration test (or, worse, never — apiservers
silently drop unknown fields by default).

## When dynamic is still the right answer

Typed isn't free. You take on:

- A dependency on the operator's Go module.
- Codegen on every type change (`make generate manifests`).
- An import cycle if the operator depends on the api back.

If any of these hold, stick with the dynamic client:

- **You don't own the CRD.** Some platform team controls the schema and hasn't
  published Go types — dynamic + a hand-written GVR is faster than negotiating a
  module release.
- **You handle arbitrary CRDs.** A generic policy engine, an admission webhook
  that mutates anything, a `kubectl`-like CLI — these get a _list_ of GVRs at
  runtime and have nothing to import.
- **You're prototyping.** If the CRD shape is in flux daily, regenerating Go
  types every change can stall iteration. Once it stabilizes, switch.

Webhookd doesn't fit any of these — it owns the SAMLGroupMapping shape via a
partner team's operator project (wiz-operator), the shape stabilized during
Phase 2 design, and the api ships against a real Go module. So we go typed, and
DESIGN-0002 / IMPL-0002 capture that decision.

## Recap — what moved

| Concern                   | Part 1 (dynamic)                                                             | Part 2 (typed)                                    |
| ------------------------- | ---------------------------------------------------------------------------- | ------------------------------------------------- |
| Source of CRD shape       | Hand-written `crd.yaml`                                                      | `note_types.go` + `make manifests`                |
| Source of Go types        | None — `map[string]interface{}` per call                                     | `note-operator/api/v1alpha1/`                     |
| Client constructor        | `dynamic.NewForConfig(cfg)`                                                  | `client.New(cfg, client.Options{Scheme: scheme})` |
| Resource identity         | `schema.GroupVersionResource{...}` package var                               | Inferred from registered Go type                  |
| CR construction           | `unstructured.Unstructured{Object: map[string]interface{}{...}}`             | `&notev1alpha1.Note{Spec: NoteSpec{...}}`         |
| Create call               | `s.dyn.Resource(gvr).Namespace(ns).Create(ctx, obj, metav1.CreateOptions{})` | `s.k8s.Create(ctx, note)`                         |
| Compile-time field safety | None                                                                         | Full                                              |
| New module dependency     | None                                                                         | `note-operator` + `controller-runtime`            |

The note-api **shrunk by ~20 lines** and gained type safety; the cost is a
sibling project we'd have built anyway if we had operator logic to ship.

## Coming in Part 3 — productionize

`03-productionize.md` (next walkthrough) takes the typed `note-api` from this
part and adds the operational layers a real service needs:

- **`log/slog`** structured logging with JSON output and request-id correlation,
  so a single request is greppable across api logs and apiserver audit logs.
- **Graceful shutdown.** `http.Server.Shutdown(ctx)` on SIGTERM, drain in-flight
  requests within a budget, fail-fast if exceeded.
- **Proper error handling.** Typed errors, classification of K8s API errors
  (`IsForbidden` → 403, `IsInvalid` → 422, transient → 503), correct HTTP status
  mapping. No more `http.StatusInternalServerError` for everything.
- **Prometheus metrics** on a separate admin port (so a leaked metrics endpoint
  can't be reached from the public network).
- **OpenTelemetry traces** exported via OTLP to a local collector.
- **Grafana** with a pre-provisioned dashboard backed by Prometheus and
  **Tempo** (for traces).
- A **`docker-compose.yaml`** wiring api + collector + Prometheus + Grafana +
  Tempo into one local stack you can `docker compose up`.

Each layer is small in isolation; the value is in composition. The typed
`note-api` you wrote in this part is the substrate for all of it — Part 3
doesn't change `note-operator` at all.
